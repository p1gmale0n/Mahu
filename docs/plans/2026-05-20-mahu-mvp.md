# Mahu MVP

## Overview

- Implement the first working MVP of Mahu, a native macOS break-reminder app.
- The MVP creates a menu-bar-only app that starts a 20-20-20 work timer on launch and shows a fullscreen break overlay on every active display when the work interval ends.
- The break overlay is dark and minimal, shows `Время отвлечься`, displays a countdown, and includes `Skip`.
- Settings are read from `~/Library/Application Support/Mahu/config.json`; missing or invalid config falls back to 20 minutes of work and 20 seconds of break.
- The implementation must keep app lifecycle/status item, config, timer, and overlay responsibilities separated so deferred features can be added without rewriting the core flow.

- **Unit tests**: required for every task that adds or changes code.

- **Timer tests**: verify work-to-break transition, break countdown, automatic return to work after break, and `Skip` behavior.
- **Coordinator tests**: use fakes for config/timer/overlay where possible to verify wiring without showing real windows.
- **Overlay tests**: keep AppKit window behavior thin and test deterministic logic through small abstractions such as display/window factories or view models.
- **E2E tests**: none exist yet. Do not invent a UI automation framework for MVP. Use manual overlay verification in Post-Completion.

## Technical Details

- Config path: `~/Library/Application Support/Mahu/config.json`.
- Config format:

```json
{
  "workDurationSeconds": 1200,
  "breakDurationSeconds": 20
}
```

- Default config: `workDurationSeconds = 1200`, `breakDurationSeconds = 20`.
- Missing config behavior: create parent directory, write default config, continue running.
- Invalid config behavior: use defaults and continue running; do not block app launch.
- Suggested module boundaries:
  - `MahuApp`: SwiftUI app entrypoint.
  - `AppDelegate`: AppKit lifecycle bridge and coordinator startup.
  - `StatusItemController`: `NSStatusItem` ownership and `Quit` menu.
  - `AppConfig`: Codable config value.
  - `ConfigStore`: Application Support file loading/writing.
  - `BreakTimer`: deterministic work/rest state machine.
  - `BreakOverlayManager`: AppKit overlay window ownership.
  - `BreakOverlayView`: SwiftUI break screen.
  - `AppCoordinator`: thin wiring layer between config, timer, status item, and overlay.
- Keep AppKit side effects at the edges. Timer/config logic should be testable without running the app or showing windows.
- Use only public APIs and avoid global input capture to preserve App Store compatibility options.
